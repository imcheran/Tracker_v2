## 🔥 Deployment Steps

### 1. **Install Firebase CLI** (if not already)
```bash
npm install -g firebase-tools
firebase login
```

### 2. **Deploy Firestore Security Rules**
```bash
# From project root:
firebase deploy --only firestore:rules
```

Verify in [Firebase Console](https://console.firebase.google.com):
- Go to Firestore > Rules
- Should show rules restricting access to couple members

### 3. **Test Partner Linking Flow**

**Setup**: Two browser windows or devices with different accounts

**Step 1 - User A generates code**:
1. Sign in as User A
2. Open Settings
3. Tap "Generate Invite Code"
4. Copy the code (e.g., `ABC1D2E3F4G5`)
5. Share with User B

**Step 2 - User B enters code**:
1. Sign in as User B
2. Go to Couples view
3. Tap "Link partner" → "I have a code"
4. Paste the code
5. Tap "Link Partner"

**Step 3 - Verify linking**:
- ✅ Both users see "You & [Partner Name]"
- ✅ Partner's timezone/status visible
- ✅ Can add shared challenges
- ✅ Real-time updates when one edits couple data

### 4. **Test Security Rules**

**Verify access control**:
```javascript
// In browser console (Firebase SDK):

// ✅ Should work - member can read couple data
firebase.firestore().collection('couples').doc(coupleId).get()

// ❌ Should fail - unauthorized user
firebase.firestore().collection('couples')
  .where('members', 'array-contains', 'OTHER_UID')
  .get()
```

## 🔧 Troubleshooting

### npm install still failing?
```bash
# Clear and retry with timeout
npm cache clean --force
npm install --ignore-scripts --legacy-peer-deps --timeout=120000
```

### Build fails after install?
```bash
# Check Vite is importable
node -e "console.log(require.resolve('vite'))" 2>/dev/null || echo "Install still running"
```

### Firestore rules won't deploy?
```bash
# Verify rules syntax
firebase serve --only firestore:rules

# Check errors in logs
firebase deploy --only firestore:rules --debug
```

### Partner linking returns "User not authenticated"?
1. Verify both users are signed in with Firebase
2. Check `user?.uid` is not null in App.tsx
3. Review browser console for auth errors

### Couple data not syncing in real-time?
1. Check Security Rules are deployed
2. Verify listener setup in App.tsx line 380+
3. Check `isCoupled && coupleId` in user doc
4. Look for `subscribeToCoupleData` errors in console

## 📚 Data Structure Reference

```firestore
users/{uid}
├── displayName, email, photoURL
├── isCoupled: boolean
├── partnerUid: string
└── coupleId: "uid1_uid2"

partnerInvites/{code}
├── senderUid, used, createdAt, expiresAt
└── (generated by generateInviteCode)

couples/{coupleId}
├── user1, user2
├── members: [uid1, uid2]  ← Used by Security Rules
├── createdAt, status
├── habits/{habitId}
├── photos/{photoId}
├── journal/{id}
└── activity/{id}
```
